# Telegram-бот консультант по портфолио и услугам

Минимальный каркас Telegram-бота на Python. Бот отвечает на `/start`, показывает меню и базовые разделы: опыт, проекты, услуги, контакты/заявка.

## Что внутри

- `bot.py` — код бота.
- `requirements.txt` — зависимости.
- `.env.example` — пример файла с переменными окружения без реальных секретов.
- `PROCESS_LOG.md` — журнал сборки, ошибок и решений.
- `BOT_PASSPORT.md` — заполненный паспорт бота по шаблону.
- `Паспорт бота.docx` — сгенерированная docx-версия паспорта, если файл присутствует после сборки.

## Как создать бота в Telegram

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте команду `/newbot`.
3. Укажите название и username бота. Username должен заканчиваться на `bot`, например `portfolio_consultant_bot`.
4. Скопируйте токен, который выдаст BotFather.

## Настройка токена

Скопируйте пример окружения:

```cmd
copy .env.example .env
```

Откройте `.env` и замените значение:

```env
TELEGRAM_BOT_TOKEN=ваш_реальный_токен_от_BotFather
CONTACT_URL=https://t.me/your_username
```

Не публикуйте `.env` и не отправляйте токен третьим лицам.

## Установка зависимостей

Рекомендуется использовать виртуальное окружение:

```cmd
python -m venv .venv
.venv\Scripts\activate
python -m pip install -r requirements.txt
```

## Запуск

```cmd
python bot.py
```

После запуска откройте своего бота в Telegram и отправьте:

```text
/start
```

Бот должен ответить приветствием и показать кнопки меню.

## Локальная проверка без токена

Проверка синтаксиса:

```cmd
python -m py_compile bot.py
```

Проверка ожидаемой ошибки конфигурации без токена:

```cmd
python -c "from bot import build_application; build_application()"
```

Если токен не задан, команда должна завершиться сообщением о необходимости `TELEGRAM_BOT_TOKEN`.
